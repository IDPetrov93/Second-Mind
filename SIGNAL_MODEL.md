# SIGNAL MODEL

## Purpose

This document defines how information enters KOS.

Every piece of incoming information begins as a Signal.

Signals initiate investigation.

They never directly modify Knowledge.

---

# Definition

A Signal is an observable indication that the state of one or more Topics may have changed.

Signals are hypotheses.

Not facts.

---

# Characteristics

Every Signal should preserve:

- original source
- timestamp
- acquisition method
- original content
- media type
- provenance

Signals are immutable.

---

# Lifecycle

Detected

↓

Validated

↓

Investigated

↓

Archived

Signals are temporary.

Knowledge is persistent.

---

# Signal Types

Examples include:

- News
- RSS
- Filing
- Social media
- Scientific publication
- Government release
- Financial report
- User submission

Signal types describe origin.

Not credibility.

---

# Core Principle

Signals never become Knowledge.

Signals only trigger Knowledge Maintenance.
