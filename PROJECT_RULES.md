# Project Rules

## Rule 1 – Architecture First
No implementation before design.

---

## Rule 2 – No unnecessary features
If a feature does not improve core understanding or decision quality, it is rejected.

---

## Rule 3 – Everything must be modular
No tightly coupled systems.

---

## Rule 4 – Explain every decision
No "because it's standard" answers.

---

## Rule 5 – Prefer boring technology
Stable > trendy.

---

## Rule 6 – Small steps only
Every module must be independently understandable.

---

## Rule 7 – No hidden complexity
No magic behavior without explanation.

---

## Rule 8 – The system must survive without any output plugins
Core must function alone.
